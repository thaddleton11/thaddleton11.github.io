Glasgows Event App

Go Back

The app had been produced in order to offer a range of digital services to attendees whilst at conferences. It is available for Android and iOS devices but a web version is offered as an alternative but, inherently, looses out on some native functionality. Each app opens Glasgows’ web-based delegate area. Here, event attendees have access to various resources; this includes programs, attendee lists, speakers, downloads and several other services.

The mobile application itself has been developed natively using Java and Swift for Android and iOS, respectively and utilized web views. This option was chosen due to the performance differences of running our web pages when compared to cross-platform hybrid available at the time. Both apps connect to Glasgows’ business system, GEM, where event specific content is loaded into a HTML based user panel template. Please read more about GEM, here.

Social Wall

The app offers a social wall to allow attendees the chance to share opinions, thoughts and photos with the rest of the conference attendees. This was implemented by applying the Bootstrap 3 template’s styling and AJAX requests which interfaces with a PHP Slim 2 backend and MySQL database. By using AJAX requests and subsequent jQuery functions, we can improve the user experience by performing changes smoother and faster without reloading the page.  

Users have the ability to upload a photo with their post from the devices camera. To allow this, the mobile app accesses the devices camera or gallery. This is initiated by using Javascript to interface with native functions. Once the user has taken or selected an image from the gallery, the chosen file is returned calling a Javascript function on the web page. This, in turn, injects the image to the waiting HTML form and the user can post the form to our server.

Once received, the image is validated before being resized using GD library and uploaded with Codeguys Upload. A problem faced was from uploaded iOS images taken in portrait. Rather than taking the photo in portrait it remains landscape and flips the image 90o. This required reading the EXIF data of the photo and detecting the rotation applied by Apple. On finding this, the image is rotated the correct way and saved without the original EXIF information.

QR code scanning

To improve the efficiency of the registration desk at events, QR code scanning has been introduced for event staff. To implement this, it would require native Android and iOS functionality, again, interfacing with frontend Javascipt. For iOS, this process was carried out similarly to accessing and returning photos from the camera as they natively offer a QR/barcode scanner. For Android, a third party library, ZXing, was required. The challenge was to initial the QR code scanner intent and return the result to the web view so the QR code can validated and feedback provided via the web page.

Redevelopment

This version of the app is to be superseded by an updated NodeJS, with Express, version. The front end will consist of a modern, responsive Bootstrap 4 mobile template, utilizing jQuery and WebSockets to enable live content changes. This enables us to do various actions that will improve the user experience during interactive sessions at events.

One scenario is live quizzes or voting. When a question is announced during a conference, delegates have to access the right menu item before registering a small amount of details, if it is their first time voting. This is, unfortunately, due to the voting software being a separate NodeJS application. By developing a NodeJS event app, this service can be integrated fully, removing the need for registration and allow us to push the question directly onto a user’s screen. This provides a better app experience, improves user engagement and reduces time wasted trying to complete the above tasks.

The revised app will interface with a RESTful API built using PHP Slim 4. The API will be a part of the GEM system that is, again, being redeveloped as the codebase is aging. From lessons learnt from developing the current version, step are being taken to ensure code is reusable and easily extendable. By using SOLID principles and design patterns like the decorater and repostory patterns.
