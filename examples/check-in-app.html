Guest Check In iPad Application

Go Back

In order to digitally indicate the arrival of an attendee at an event, an in-house iPad app was created to solve this problem. The stipulations were it must work with multiple app operating at the same event and be able to continue working when an internet connection is not available.

For the iOS app, the Glasgows Event App** was repurposed and rebranded appropriately. To perform the task itself, the app serves locally stored HTML, a necessity for offline use. The list of attendees is displayed via Bootstrap 3 and DataTables, a jQuery plug-in that paginates results and allows searching against the contents of the table. Modifications were made to DataTables to allow the content to be build through AJAX requests to the API and to add an addition column that would house a user hash and tick box that would begin registering a guest as arrived.

On each request, the API returns any changes that occurred after the last API key was issued to the app. This now will show the user the updated attending statuses of guests that have been performed by other staff members using the app.

To accommodate the offline functionality, IndexedDB client-side database was used to assist. Each API request is processed and saved into the database. When request fails to communicate with the server, the user is notified of the offline status, any further changes are saved locally and the app can continue to operate. Once a connection can be established again, any locally stored changes are pushed updating the remote database.

Accompanying the frontend is a PHP Slim 3 backend, with a MySQL and Doctrine ORM, which provides the API and administration system.  The Bootstrap 4 admin system allows users to create new and edit guest lists, an associated login and to set configurations like a custom logo, light or dark mode, column names and columns to hide/display. There are also options to determine access dates and times and allow clients access to a slimmed down basic version. At anytime during the event, both admins and clients are able to download a CSV of the results if they so desire.
